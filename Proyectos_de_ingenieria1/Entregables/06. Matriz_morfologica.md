## 6. 🧩 **Matriz Morfológica** 

Sistema de alerta temprana de heladas y sequías – Prototipo MVP
La matriz morfológica descompone el sistema WILLAY en sus funciones principales y explora las posibles soluciones técnicas para cada una.

### **Tabla 1: Matriz Morfológica del Sistema WILLAY**

<img width="1536" height="1024" alt="image" src="https://github.com/user-attachments/assets/2fbfa187-64fc-4669-8069-12dae9ee402b" />



La selección de la configuración del sistema se realizó considerando los siguientes criterios:

- ## **Viabilidad económica**

Se priorizaron soluciones de bajo costo compatibles con un entorno académico.

- ## **Facilidad de implementación**

Se seleccionaron tecnologías con alta disponibilidad, documentación y facilidad de integración.

- ## **Generación de datos útiles**

El sistema debe permitir recolectar datos reales que puedan utilizarse posteriormente en modelos predictivos.

- ## **Escalabilidad**

La arquitectura debe permitir futuras mejoras sin rediseño completo.

- ## **Coherencia técnica**

Se evitaron combinaciones incompatibles entre transmisión, procesamiento y energía.

**Justificación de Selección:**
Justificación de selección

La configuración elegida responde a la necesidad de desarrollar un prototipo funcional que valide el sistema sin introducir complejidad innecesaria.

Se prioriza:

- Captura de variables críticas (temperatura y humedad)
- Generación de datos en tiempo real
- Visualización inmediata mediante indicadores físicos
- Uso de infraestructura cloud como base futura

Adicionalmente, el sistema no se limita a un único modelo de inteligencia artificial, sino que se plantea una evaluación comparativa entre diferentes enfoques (regresión, Random Forest y redes neuronales), permitiendo seleccionar el más adecuado en función de los datos obtenidos.

**Configuración seleccionada del sistema**

| Función        | Selección final                            |
| -------------- | ------------------------------------------ |
| Captura        | Temperatura y humedad (DHT22)              |
| Adquisición    | Lectura periódica (30–60 s)                |
| Transmisión    | Visualización local + opción futura a nube |
| Almacenamiento | Cloud (Firebase / AWS)                     |
| Procesamiento  | Análisis básico + nube                     |
| Modelado       | Evaluación de múltiples modelos            |
| Alertas        | LEDs                                       |
| Interfaz       | Indicadores físicos                        |
| Energía        | Conector 5V                                |

## Aclaraciones técnicas importantes
- Transmisión de datos: Se refiere al destino de la información (local, teléfono, nube).
- Almacenamiento: Se realiza exclusivamente en la nube (Firebase, AWS u otros).
- Modelado: No se fija un modelo único; se evaluarán múltiples enfoques.
- Energía: Se utiliza alimentación por conector de 5V para simplificar el prototipo.


La matriz morfológica permitió estructurar el diseño del sistema WILLAY de manera integral, facilitando la selección de una configuración que equilibra costo, funcionalidad y viabilidad técnica.

El prototipo desarrollado no solo valida el concepto de monitoreo climático, sino que establece una base sólida para la implementación futura de modelos predictivos aplicados a la agricultura andina.
